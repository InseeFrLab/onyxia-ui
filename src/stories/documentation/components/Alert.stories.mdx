# Alert

This is a documentation of `<Alert/>` component.
Alerts are used to draw the user's attention to information.

-   [Structure](#structure)
-   [Types of alert](#types-of-alert)
-   [Rules of use](#rules-of-use)
-   [Properties](#properties)

## Structure

The alert is made up of the following elements:

-   a title - mandatory.

-   a pictogram and a color - mandatory.

-   a description text - optional.

-   a closing cross - optional.

Alerts are displayed contextually in a page / form, during user interactions with validation messages
(example: success or error following submission of a form) or during application / system side events
(example : information, alert, update messages, etc.).

## Types of alert

There are 4 types of alert allowing information of different kinds to be given:

-   [Error alert](#error-alert)
-   [Success alert](#success-alert)
-   [Information alert](#information-alert)
-   [Warning alert](#warning-alert)

### Error alert

Used when there are several errors in a form, or fatal errors for the user to report.

## Rules of use

-   The title of the alert should be clear and concise.
-   The alert description text should clearly detail the information / problem to the user.
-   The tone should be courteous, not to blame the user, but to accompany him.
